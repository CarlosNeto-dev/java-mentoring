Sintaxe Básica do Java 🏗️

Neste capítulo, saímos da teoria de instalação e mergulhamos na Sintaxe Básica, o verdadeiro "DNA" que sustenta qualquer aplicação Java robusta. Dominar esses conceitos é o que separa um código amador de um sistema de nível Fellow.

  • ⚙️ 2.2 Operadores: O motor da lógica computacional. Vamos explorar desde cálculos matemáticos precisos até expressões lógicas complexas (&&, ||, !) que permitem ao seu software realizar comparações e tomar decisões rápidas.

Variáveis e Tipos de Dados 📦

No Java, uma variável é um espaço na memória do computador onde guardamos informações. Imagine como uma caixa com uma etiqueta: a etiqueta é o nome da variável, e o formato da caixa é o tipo de dado.

🏗️ Como declarar uma variável? 


int idade = 18; // Declarando um número inteiro

🧪 Os Tipos Primitivos (Os Essenciais):

Existem tipos básicos que você vai usar 99% do tempo:

  • Inteiro 🔢: 
    
public class Main {
  public static void main(String[] args) {
    int number1 = 20;
    int number2 = 10;

    int result = number1 + number2;

    System.out.println(result); // Saída: 30
  }
}
  • Flutuante 🔢: 
    
public class Main {
  public static void main(String[] args) {
    double number1 = 2.5;
    double number2 = 2.5;

    double result = number1 + number2;

    System.out.println(result); // Saída: 5.0
  }
}
  • Booleano 🔀: 
    
public class Main {
  public static void main(String[] args) {
    boolean isTrue = true;
    boolean isFalse = false;

    System.out.println(isTrue); // Saída: true
    System.out.println(isFalse); // Saída: false
  }
}
  • char 📝: 
    
public class Main {
  public static void main(String[] args) {
    char letter = 'A';

    System.out.println(letter); // Saída: A
  }
}
⚠️ Atenção: Existe também o tipo String, que usamos para textos (como seu nome "Carlos", "Maria" etc...). No Java, String não é um tipo primitivo, mas sim uma Classe, por isso sempre começa com S maiúsculo! ⚠️

📏 Regras de Ouro para Nomes de Variáveis

Para manter o seu código no padrão Clean Code:

  • CamelCase🐫: Comece com letra minúscula e cada palavra seguinte com maiúscula (ex: minhaVariavel, valorBolsaEstagio).
  • Seja descritivo📔: Nada de criar variáveis chamadas a, b ou x. Use nomes que expliquem o que o dado é (ex: salarioMensal).

Operadores Aritméticos e Lógicos ➕➖

Os operadores aritméticos são usados para realizar operações matemáticas simples. O Java segue a regra de precedência (PEMDAS), ou seja, multiplica e divide antes de somar e subtrair.

  • Divisão Inteira(/): Se você usar números inteiros, o Java vai descartar o que vem depois da vírgula. 🪓
  • Módulo(%): Ele não te dá o resultado da divisão, mas sim o resto que sobrou dela. É perfeito para verificar se um número é par ou ímpar! ⚖️

Matemática na Prática:

Veja como o Java se comporta com números inteiros e como o resto da divisão funciona. 🧮


public class Main {
  public static void main(String[] args) {
    int number1 = 10;
    int number2 = 3;

    int division = number1 / number2;
    int restOfDivision = number1 % number2;

    System.out.println(division); // Saída: 3

  }
}

Operadores de Comparação (Relacionais) ⚖️

Esses operadores servem para comparar dois valores. O resultado de qualquer comparação no Java será sempre um valor booleano: true ou false.

  • == : Verifica se os valores são iguais (não confunda com =, que serve para dar valor).
  • != : Verifica se são diferentes.
  • > e < : Maior e Menor. 📈

Comparando Valores

As comparações são a base para que o seu programa tome decisões no futuro. 🚦 


public class Main {
  public static void main(String[] args) {
    int minAge = 18;
    int userAge = 20;

    boolean canDrive = userAge >= minAge;
    boolean isDiferent = (10 != 5);

    System.out.println(canDrive); // Saída: true
    System.out.println(isDiferent); // Saída: true
  }
}

Operadores Lógicos 🧠

Eles permitem combinar várias comparações em uma única expressão. São fundamentais para criar regras complexas.

  • && (E / AND): Retorna true apenas se ambas as condições forem verdadeiras. ✅ + ✅
  • || (OU / OR): Retorna true se pelo menos uma das condições for verdadeira. ✅ + ❌
  • ! (NÃO / NOT): Ele inverte o valor. Se algo é true, vira false. 🔄

Lógica de Acesso

Usando lógica para verificar se alguém pode entrar em um evento VIP. 🎫 


public class Main {
  public static void main(String[] args) {
    boolean haveInvite = true;
    boolean isOnList = false;

    boolean canEnter = haveInvite || isOnList;

    boolean vipAcess = haveInvite && isOnList;

    System.out.println(canEnter); // Saída: true
    System.out.println(vipAcess); // Saída: false
  }
}

Controle de Fluxo: Condicionais e Loops🚥

O if (se) é a estrutura de decisão mais básica. Ele avalia uma condição booleana: se for true, o código dentro das chaves executa. Se não, o Java pula para o else (senão).

  • if: A porta de entrada. Se a condição for real, ele entra.
  • else if: Uma segunda chance. Se o primeiro if falhar, ele testa essa nova condição.
  • else: O "porto seguro". Se nenhuma das condições acima funcionar, ele executa o que estiver aqui.

Tomando Decisões:

Um sistema simples para verificar se um aluno foi aprovado, está de recuperação ou reprovado. 📝 


public class Main {
  public static void main(String[] args) {
    double grade = 6.5;

    if (grade >= 7) {
      System.out.println("Aprovado! 🎉");
    }

    else if (grade >= 5) {
      System.out.println("Recuperação! 📚");
    }

    else {
      System.out.println("Reprovado! ❌");
    }
  }
}

O Laço for (Repetição Controlada) 🔄

O for é usado quando você sabe exatamente quantas vezes quer repetir uma ação. Ele é composto por três partes: inicialização, condição e incremento.

  • Início: int i = 0 (Começamos do zero).
  • Condição: i < 5 (Enquanto i for menor que 5, continue rodando).
  • Incremento: i++ (A cada volta, some +1 no valor de i)

Contagem Inteligente

Usando o laço para repetir uma mensagem ou percorrer uma sequência de números. 🚀 


public class Main {
  public static void main(String[] args) {
    // Vai imprimir de 0 a 5
    
    for (int i = 0; i <= 5; i++) {
      System.out.println("Count:" + i);
    }

    System.out.println("Done!");
  }
}

O Laço while (Repetição por Condição) 🔁

Diferente do for, o while (enquanto) é usado quando você não sabe quantas vezes a repetição vai acontecer. Ele depende exclusivamente de uma condição ser verdadeira para continuar rodando.

⚠️ Cuidado: Se a condição nunca se tornar falsa, você cria um "loop infinito" e o programa trava!

Enquanto for Verdade...

Simulando o carregamento de uma bateria até que ela chegue a 100%. 🔋 


public class Main {
  public static void main(String[] args) {
    int battery = 95;

    while (battery < 100) {
      System.out.println("Battery: " + battery + "%");
      battery++;
    }

    System.out.println("Charge Complete! 🔋✅");
  }
}

O Laço do-while 🔁❓

Diferente do while comum, que testa a condição no início, o do-while testa no final. É perfeito para situações onde você precisa que uma ação ocorra obrigatoriamente antes de qualquer validação (como exibir um menu pelo menos uma vez).

⚠️ O Perigo: Se você não tomar cuidado, ele executa algo que não deveria ter sido executado caso a condição já fosse inválida.

Garantindo a Primeira Execução

Veja como o código roda uma vez, mesmo que a bateria já esteja "estourada" acima de 100%. 🔋 


public class Main {
  public static void main(String[] args) {
    int battery = 120; // Já passou do limite!

    do {
      System.out.println("Battery: " + battery + "%");
      battery++;
    } 
    
    while (battery < 100);

    // No 'while' comum, nada seria impresso. 
    // No 'do-while', ele imprime "Carga atual: 120%" e depois para.
  }
}

Métodos e Funções

O que é um Método? 🛠️

Pense em um método como uma ferramenta ou uma receita. Em vez de escrever a mesma lógica dez vezes, você cria um método uma única vez e o "chama" sempre que precisar.

Um método básico em Java tem:

  • Tipo de Retorno: O que ele devolve (ex: int, String, ou void se não devolver nada).
  • Nome: O que ele faz (ex: somar, imprimirNome).
  • Parâmetros: O que ele precisa para funcionar (fica dentro dos parênteses).

Criando sua primeira Ferramenta

Neste exemplo, criamos um método que recebe um nome e imprime uma saudação personalizada. ✋ 


public class Main {
  // Criando o método (A Receita)
  public static void greet(String name) {
    System.out.println("Hello, " + name + "!");
  }

  // Chamando o método (Usando a ferramenta)
  public static void main(String[] args) {
    greet("Carlos");
    greet("Maria");
  }
}

Métodos com Retorno (return) 🔄

Às vezes, você não quer apenas que o método imprima algo, mas que ele calcule e te devolva um valor para você usar depois. Para isso, usamos a palavra-chave return.

  • Se o método devolve um número inteiro, o tipo deve ser int.
  • Se ele não devolve nada, usamos void.

Métodos que Calculam

Aqui o método faz uma conta e "entrega" o resultado para uma variável fora dele. 🧮 


public class Main {
  public static int sum(int n1, int n2) {
    return n1 + n2;
  }

  public static void main(String[] args) {
    int result = sum(10, 2);
    System.out.println(result); // Saída: 12
  }
}

Tratamento de Erros e Exceções

O que são Exceções? ⚠️

No Java, um erro que acontece enquanto o programa está rodando é chamado de Exceção. Imagine que você tenta dividir um número por zero ou ler um arquivo que não existe: o Java entra em pânico e para tudo.

O try-catch é como um "seguro" para o seu código: você tenta fazer algo e, se der erro, você captura o problema e decide o que fazer em vez de deixar o programa fechar.

Evitando o Crash do Sistema

Veja como capturamos um erro matemático clássico para manter o programa rodando. 🛡️ 


public class Main {
  public static void main(String[] args) {
    try {
      int result = 10 / 0; // Isso gera um erro!
      System.out.println(result);
    } 
    
    catch (ArithmeticException e) {
      // O código vem para cá se houver erro matemático
      System.err.println("ERRO: Não é possível dividir por zero!");
    }

    System.out.println("O programa continua rodando normalmente... ✅");
  }
}

O bloco finally 🏗️

O finally é um bloco opcional que vem depois do catch. O código dentro dele sempre será executado, não importa se deu erro ou se deu tudo certo. É muito usado para "limpar a bagunça", como fechar uma conexão com banco de dados ou um arquivo. 


public class Main {
  public class void main(String[] args) {
    try {
      System.out.println("Tentando abrir o arquivo...📂");
      // Lógica aqui 
    }

    catch (Exception e) {
      System.out.println("Houve um erro inesperado. ❌");
    }

    finally {
      System.out.println("Fechando recursos e finalizando processo. 🔒");
    }
  }
}

Programação Orientada a Objetos (POO)

O que é uma Classe e um Objeto? 🏷️

Para facilitar sua vida, use a analogia da Planta de uma Casa:

  • Classe: É o projeto (a planta) no papel. Ela diz que a casa deve ter janelas, portas e uma cor, mas a planta em si não é uma casa onde você pode morar. 📝
  • Objeto: É a casa construída na vida real usando aquela planta. Você pode construir várias casas (objetos) diferentes (uma azul, uma amarela) usando o mesmo projeto (classe). 🏠

Atributos e Métodos:

  • Atributos: São as características (cor, tamanho, modelo). 🎨
  • Métodos: São as ações (abrir porta, ligar luz). ⚡

Criando sua primeira Classe 🛠️

Em Java, uma classe geralmente fica em um arquivo separado. Vamos criar uma classe simples para representar um Celular.

Definindo o Projeto (Classe)

Aqui definimos quais características todo celular terá e o que ele pode fazer. 📱 


public class Phone {
  // Atributos (Características)

  String mark;
  String model;
  int battery = 100;

  // Métodos (Ação)
  void connect() {
    System.out.println("Phone" + model + "Connecting...");
  }
}

Instanciando um Objeto (Dando vida ao código) 🌟

Depois de criar a planta, precisamos "construir" o objeto no nosso método main usando a palavra-chave new.

Criando o Objeto Real

Agora transformamos o projeto em algo real na memória do computador. 🚀 


public class Main {
  public static void main(String[] args) {
    // Criando o objeto myPhone
    Phone myPhone = new Phone();

    // Atribuindo valores
    myPhone.mark = "Apple";
    myPhone.model = "Iphone 15";

    // Chamando um método
    myPhone.connect();
  }
}

O Grande Insight: O new e o Apelido do Clone 👤

Para fechar o capítulo, pense no processo de criar um objeto da seguinte forma:

  • A Classe é o DNA: Ela contém todas as instruções de como algo deve ser, mas ela sozinha não "existe" no mundo real. 🧬
  • O new é a Máquina de Clonagem: Quando você digita new, o Java vai lá no DNA (Classe) e cria uma cópia exata dele na memória do computador. 🏗️
  • O Objeto é o "Apelido": A variável que você cria (ex: meuCelular) é apenas o apelido que você dá para aquele clone específico.
